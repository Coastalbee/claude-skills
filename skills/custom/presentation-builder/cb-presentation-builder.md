# Presentation Builder

Generate complete, polished HTML slide presentations from video scripts, outlines, or topic briefs. Output is a single `.html` file with embedded CSS and JS — no dependencies except Google Fonts.

## Dependency

Follow the **frontend-design** skill principles: distinctive design, no generic AI aesthetics, bold typography, intentional color choices. Every presentation should feel custom-designed for its topic.

## Reference Files

- `references/design-system.md` — CSS variables, typography, component styles extracted from a proven presentation. Use as **inspiration**, not a rigid template. Create fresh palettes and typography per presentation.
- `references/slide-templates.md` — HTML patterns for every slide type with content limits.
- `assets/template.html` — Starter HTML with all boilerplate (CSS, JS, navigation). Copy and fill in slides.

## Process

### 1. Analyze the Input

Read the script/outline/topic and identify:
- The core narrative arc (what's the story?)
- Key sections and transitions
- Data points, comparisons, lists, quotes
- How many slides are needed (aim for 10–20)

### 2. Map Content to Slide Types

Each piece of content maps to a slide type. Choose based on what the content IS:

| Content | Slide Type |
|---------|-----------|
| Opening / hook | **Title** |
| Bold claim, thesis, or quote | **Quote/Statement** |
| Multiple items with descriptions | **Content Grid** (cards) |
| Before/after, old/new, pros/cons | **Comparison** (two columns) |
| File structures, hierarchies | **Tree/Structure** |
| Systems with connections between parts | **Diagram** (multi-element) |
| Features, tools, plugins | **Plugin/Feature Cards** |
| Numbers, metrics, results | **Stats/Results** (big numbers) |
| Sequential instructions, how-to | **Steps/Numbered List** |
| Visual evidence, UI showcase | **Screenshot Showcase** |
| Final pitch, links, subscribe | **CTA/Closing** |

### 3. Design Decisions

Before writing HTML, decide:

- **Color palette**: Pick 1 accent color + 1-2 supporting colors that match the topic. Use CSS custom properties.
- **Typography**: Choose from Google Fonts. Pair a **display font** (headings) with a **mono font** (labels, code, counters). Avoid Inter, Roboto, Arial, system fonts. Good choices: Sora, Space Grotesk, Outfit, Instrument Sans, Plus Jakarta Sans paired with JetBrains Mono, Fira Code, IBM Plex Mono.
- **Mood**: Dark theme by default. Adjust surface colors and glow effects to match topic energy.

### 4. Build the Presentation

Start from `assets/template.html`. For each slide:
1. Pick the HTML pattern from `references/slide-templates.md`
2. Fill in content
3. Add `animate` and `delay-N` classes for staggered reveals
4. Set the slide counter `NN / TOTAL`
5. **Check: does it fit in one viewport?** If not, split into multiple slides.

### 5. Verify

- Every slide fits in 100vh — no scrolling within a slide
- Screenshots get their own dedicated slides
- Navigation dots work (auto-generated by JS)
- Arrow keys navigate between slides
- Animations trigger on scroll
- Slide numbers are sequential and correct
- All Google Fonts are loaded in the `<link>` tag

## Rules

1. **One viewport per slide.** Every slide must fit in 100vh × 100vw. If content overflows, split into multiple slides. No exceptions.
2. **Screenshots get their own slides.** Never put a screenshot on a content-heavy slide. Give it a label/title and full visual space.
3. **Single-file HTML.** All CSS and JS embedded. Only external dependency is Google Fonts CDN.
4. **Scroll-snap mandatory.** `html { scroll-snap-type: y mandatory; }` and `.slide { scroll-snap-align: start; }`.
5. **Keyboard navigation.** Arrow keys (up/down and left/right) move between slides.
6. **Dot navigation.** Fixed right-side dots, active state highlights current slide.
7. **Fade-up animations.** Elements animate in with staggered delays using `.animate` and `.delay-N` classes triggered by IntersectionObserver.
8. **Slide numbering.** Every slide has `NN / TOTAL` in the top-right corner using mono font.
9. **No generic aesthetics.** Each presentation must have intentional design choices — custom colors, distinctive fonts, themed accents.
10. **Content limits per slide type.** Follow the max items/text guidelines in `slide-templates.md` to prevent overflow.
